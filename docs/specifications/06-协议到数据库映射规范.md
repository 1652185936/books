| 项目 | 内容 |

| 文档编号 | MX-MAP-001 |

| 编写日期 | 2026-06-19 |

本规范是「学习包协议 JSON → 数据库」字段级映射、派生数据生成、去重与主答案选取、
错误分类词表的**唯一真值**。导入服务（`ImportService` 的 `indexing` 阶段）必须按本规范实现。
与 03-LLD §3 表结构冲突时，字段语义以本规范为准，并同步修订 LLD。

> 解决评审项：R-003、R-005、R-007、R-008、R-010、R-011、R-029（其中 R-002、R-004 已由 Codex 在
> chapter.schema / 协议 §6.3 / LLD §3.8 解决）。本规范声明的新增表/列在 03-LLD §3 与各 Schema 落地。

## 1. 版本字段映射（R-029）

| 协议字段 | 数据库字段 | 说明 |
|---|---|---|
| `manifest.formatVersion` | `books.format_version` | 协议版本，用于主版本白名单兼容判断 |
| `manifest.contentVersion` | `books.content_version` | **内容版本**，去重与升级判定键 |
| `book.schemaVersion` | 不入库 | 仅用于章节/书 JSON 的结构校验 |

- 去重/升级键 = `(package_id, content_version)`（对应 `books.UNIQUE(package_id, content_version)`、LLD §9）。
- **协议必须新增 `manifest.contentVersion`**（语义化版本字符串）。本规范同步要求 `manifest.schema.json`
  将其列为 `required`，协议 §3 示例补该字段。无 `contentVersion` 的包按缺字段拒绝（IMP-008）。
- `manifest.formatVersion` 用于 IMP-006 主版本判断；`content_version` 只做内容去重，二者不可混用。

## 2. 书籍 / 章节 / 小节映射

| 协议 | 数据库 | 备注 |
|---|---|---|
| `manifest.book.*` / `book.json.*` | `books.*` | 以 `book.json` 为内容真值，`manifest.book` 仅用于导入前预览 |
| `book.toc[]` + 章节 JSON 头 | `chapters.*` | `toc[].file` 缺失或与章节 JSON `id` 不一致 → IMP-010/IMP-011 |
| `book.source.pdfPageCount` | `books.source_page_count` | 可空，仅追溯 |
| 章节/小节 `sourcePages`、`sourceRef.pdfPages` | 对应 `source_pages_json` | 序列化为 JSON 整数数组 |

### 2.1 小节置信度（R-010）

`sections.confidence ← section.sourceRef.confidence`（直接取，二者均为单一标量，不存在多来源页取舍）。
`sourceRef` 在 chapter.schema 中为小节必填，故导入期恒有值；缺失属 Schema 校验失败（IMP-008），不进入业务层。
章节级 `quality.ocrConfidence` 只写 `chapters` 的复核相关判断，**不**下放为 `sections.confidence`。

## 3. 内容块派生数据（R-005）

协议的 block 没有 `sourcePages` 字段，且 `plain_text` 是派生值。两者均在 `indexing` 阶段一次性计算后落库，
渲染期不再重算。

### 3.1 `content_blocks.source_pages_json`

`= 所属 section.sourceRef.pdfPages`（块级来源页统一继承自所在小节）。

### 3.2 `content_blocks.plain_text` 按 `type` 派生

| block.type | plain_text 派生规则 |
|---|---|
| heading / paragraph | 取 `text` |
| formula | 拼接各 `parts[].text`，以空格分隔 |
| example | `english` + ` ` + `chinese` |
| example-list | 各 example 的 `english`/`chinese` 顺序拼接，换行分隔 |
| bullet-list | 各列表项文本换行拼接 |
| table | 表头 `columns` + 各 `rows` 单元格按行拼接（空格分隔单元格、换行分隔行） |
| timeline | 各节点标题/说明拼接 |
| callout | 标题 + 正文 |
| illustration | 取 `alt`（不含图片二进制） |
| audio-text | 取文本字段 |

- 统一标准化为单行内可搜索文本（折叠多余空白）。
- 渲染白名单以协议 §5.2 的 11 种为准；映射表见 LLD §6.5（须补 `bullet-list`、`audio-text` 两项渲染组件，归 Batch D5）。

## 4. 练习与答案映射

### 4.1 判分开关（R-026，配合 04 契约 `gradingOptions`）

协议 `exercise.grading` 的标量开关落入新增列 `exercises.grading_options_json`（Batch D4 在 LLD §3.8 落地）：

```text
caseSensitive, trimWhitespace, collapseWhitespace,
ignoreTerminalPunctuation, normalizeApostrophes, maxEditDistance
```

判分引擎（04 契约 `GradingEngine`）从 `ExerciseDetail.gradingOptions` 读取，不得硬编码默认值。
`grading.mode`（如 `grammar-aware`）写入 `grading_options_json.mode`。

### 4.2 `answers.<blankKey>` → `accepted_answers`（R-011、R-030、R-035）

1. **标准化生成**：对 `display` 和每个 `accepted[]` 套用与运行期**完全相同**的 §4.3（LLD）标准化函数，
   得到 `normalized_answer`；`display_answer` 保留协议原始串。导入端与判分端必须共用同一标准化实现。
2. **主答案选取（R-030）**：`accepted` 中标准化后等于 `normalized(display)` 的那条置 `is_primary=1`；
   若 `display` 未出现在 `accepted` 中，则把 `display` 自身作为一条主答案插入。每个 blank 恰好一条 `is_primary=1`。
3. **去重（R-035）**：标准化后 `normalized_answer` 相同的多条 `accepted` **合并为一行**（保留首个 `display_answer`，
   `sort_order` 按出现顺序递增）。这是预期行为，**不算导入失败**——解决协议自带示例
   `"'s drinking"` 与 `"’s drinking"` 归一后撞 `UNIQUE(blank_id, normalized_answer)` 的问题。

### 4.3 `tokens` → `exercise_blanks.tokens_json`

直接序列化协议 `answers.<blankKey>.tokens`。tokens 为空时落 `'[]'`。

### 4.4 `grading.partialRules[]` → `grading_rules`（R-003）

| 协议字段 | grading_rules 字段 | 映射规则 |
|---|---|---|
| 数组下标（从 1） | `priority` | 数组顺序即优先级 |
| `whenTokensEqual` | `rule_type` + `params_json` | 默认 `rule_type='tokens-equal'`，`params_json={"tokens":[...]}`；若该规则 `result≠correct` 且语义为“缺少某 token”，转换器应显式给 `ruleType='missing-token'` |
| `result` | `result` | 直接映射（correct/partial/incorrect） |
| `id` | `feedback_code` | **`feedback_code = partialRule.id`**，解决 NOT NULL 无来源 |
| `feedbackZh` | `feedback_zh` | 直接映射 |

- 协议 partialRule 可选携带 `ruleType` 字段以消除 `tokens-equal` / `missing-token` 歧义；缺省按上表。
- 内置语法规则（LLD §4.4 第 5 步）使用本规范 §5 的固定 `feedback_code`，不来自学习包。

## 5. 反馈与错误分类受控词表（R-007、R-028）

`grading_rules.feedback_code`、`exercise_attempts.feedback_code`、`wrong_items.error_type` 共用下表受控词：

| code / error_type | 含义 |
|---|---|
| `missing-auxiliary` | 缺助动词（be/do/have） |
| `wrong-tense` | 时态错误 |
| `wrong-form` | 词形错误（如未加 -ing/-ed） |
| `spelling` | 拼写错误 |
| `missing-word` | 缺词 |
| `extra-word` | 多词 |
| `word-order` | 词序错误 |
| `recognition-uncertain` | 编辑距离接近，疑似识别/拼写误差（仅提示，不判 correct） |
| `other` | 其他（必须附 `feedbackZh`） |

- `attempts.feedback_code` ← 命中的 `grading_rule.feedback_code`，或内置规则给出的上表 code。
- `wrong_items.error_type` ← 该次判分主导 blank 的 `feedback_code`（多空题取首个非 correct 空）。
- 学习包自定义 `partialRule.id` 若不在上表，导入时归一为 `other` 并保留原 id 于 `params_json.sourceId`。

## 6. 错题汇总规则（R-007）

`wrong_items` 按 `exercise_id` 幂等 upsert（`UNIQUE(exercise_id)`）：

- 首次判 partial/incorrect：创建行，`error_count=1`、`correct_streak=0`、`status='active'`、`error_type` 见 §5。
- 再次判 partial/incorrect：`error_count+1`、`correct_streak=0`、更新 `latest_attempt_id` 与 `error_type`。
- 重做判 correct：`correct_streak+1`；达到阈值 `correct_streak ≥ user_settings.wrongbook.masteryStreak`（默认 3）
  时 `status='mastered'`，**保留历史**（`error_count` 不清零）。
- `first_attempt_id` 一经写入不再更改；历史明细以 `exercise_attempts` 为准。

## 7. 复习卡来源映射（R-002，已落地，归档备查）

flashcard 结构与导入语义已在协议 §6.3、`chapter.schema.json.flashcards`、LLD §3.19.1 定义：
`source_type='package'`、`source_id='<bookId>:<flashcard.id>'`、`front_json/back_json ← flashcard.front/back`。
本规范不再重复，仅登记其为 D1 已闭合项。

## 8. report.json 复核映射（R-008）

`report.json.reviewQueue[]` 与 `chapter.quality` 复核信息当前只复制进包目录、未入库。规则：

- 新增表 `content_review_flags`（Batch D4 在 LLD §3 落地）：

  | 字段 | 说明 |
  |---|---|
  | book_id | FK books.id ON DELETE CASCADE |
  | chapter_external_id | 复核项所属章节外部 ID |
  | json_path | reviewQueue[].jsonPath |
  | severity | low / medium / high |
  | type | reviewQueue[].type（如 ocr-uncertain） |
  | message | 脱敏后的复核说明 |

- 导入 `indexing` 阶段把 `reviewQueue[]` 落入该表，按 `file` 解析出 `chapter_external_id`。
- **IMP-015 触发条件**（统一口径）：`reviewQueue` 非空 **或** 任一 `chapter.quality.reviewStatus='needs-review'`。
- F04 章节「需复核」标识、F05 块级复核提示读取本表。

## 9. 学习事件权威源（R-009）

避免统计双重计数，明确单一事实源：

| 指标 | 权威来源 | 写入方 |
|---|---|---|
| 有效学习时长、活跃/暂停 | `learning_events`（仅 `open`/`active`/`pause`） | `StatisticsService` / 会话计时器 |
| 练习数、正确率 | `exercise_attempts` | 判分事务 |
| 复习次数、间隔变化 | `review_logs` | `ReviewScheduler` |

- `learning_events` **不**重复记录 `answer`/`review` 明细（删除该用途），只承载时间/活跃度。
- 统计页（F11）按上表分别取数，禁止同时从 events 与 attempts 累加同一指标。

## 10. 升级时冗余列与锚点同步（R-006）

学习包升级（`package_id` 不变、`content_version` 变化）采用**按 `external_id` 原地 UPDATE**，内部主键保持稳定：

- `chapters`/`exercises`/`sections` 以 `external_id` 匹配旧行 UPDATE，不 delete+insert，
  从而 `exercise_attempts.{book_id,chapter_id,exercise_id}`、`wrong_items.exercise_id`、
  `review_cards.chapter_id` 等冗余/外键列自然保持一致，历史答题与错题不丢失（满足 FR-F03-06、验收 5）。
- 旧包存在、新包缺失的 `external_id`（被删章节/练习）：对应 `exercises` 置 `enabled=0` 并保留其 attempts/wrong_items，
  不物理删除；笔记/高亮/进度按 §11 与 LLD §3.12A/§9 的 `needs-relocation` 规则处理。
- `reading_progress.last_section_id/last_block_id`、`section_progress` 按 section `external_id` 重映射；
  无法映射时回退到 `scroll_anchor_json` 或章节首，不直接置脏。

## 11. 收藏粒度（R-012）

- `bookmarks` 增加 `scope TEXT NOT NULL CHECK IN ('chapter','block')`（Batch D4 在 LLD §3.18 落地）。
- 章节级收藏（`scope='chapter'`, `block_id IS NULL`）唯一性用部分唯一索引：
  `UNIQUE(book_id, chapter_id) WHERE scope='chapter'`，避免同章重复收藏（SQLite/relationalStore 允许多 NULL 的问题）。
- 块级收藏沿用 `UNIQUE(book_id, chapter_id, block_id)`；升级后块被删时按 §10 处理，不塌缩为章节收藏。
