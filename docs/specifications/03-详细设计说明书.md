### 1.3 通用数据约束

- 内部主键：`TEXT`，使用 UUIDv7 或等价有序唯一 ID。
- 学习包外部 ID：单独存为 `external_id`，不得直接假设跨书唯一。
- 时间：`INTEGER`，UTC Unix epoch milliseconds。
- 布尔：`INTEGER NOT NULL CHECK(value IN (0,1))`。
- 枚举：`TEXT NOT NULL CHECK(x IN (...))`，数据库 `CHECK` 与 ArkTS 联合类型双重约束。**必须带 `NOT NULL`**：SQLite 的 `CHECK(col IN (...))` 在 `col IS NULL` 时结果为 NULL（非 FALSE），NULL 会穿透 CHECK 写入，而 ArkTS 联合类型不含 `null`，读出即落空（R-039）。有自然初值的状态列应给 `DEFAULT`（如 `reading_progress.status='not-started'`、`recognition_status='not-requested'`、`grade='draft'`）。`grading_rules.blank_id` 等已声明可空的非枚举外键除外。
- JSON：`TEXT`，写入前必须解析、校验和重新序列化。
- 文件路径：只保存应用沙箱内相对路径，不保存用户文件绝对路径。
- 数据库启用外键；所有迁移必须单调递增。
- 用户数据表采用软删除时必须有 `deleted_at`，查询默认排除已删除数据。
- 协议 JSON 到数据库的字段级映射、派生数据生成、答案去重与主答案选取、错误分类受控词表，以《[06-协议到数据库映射规范](./06-协议到数据库映射规范.md)》为准；本节表结构与其冲突时以该规范字段语义为准。
- **跨层命名映射约定（R-046）**：三套风格分工固定——数据库列 `snake_case`、协议/DTO JSON 字段 `camelCase`、枚举值 `kebab-case`。DB↔JSON 字段名由统一 mapper 按 snake↔camel 规则自动转换，不在各处手写转换（避免 `titleZh`↔`title_zh` 漂移）。枚举字面量集中定义为 ArkTS `const` 联合类型，并**由同一常量来源生成数据库 `CHECK` 取值列表**，避免连字符/大小写笔误在运行期才暴露为 CHECK 失败。
- **尺寸单位与换算（R-035）**：布局/触控/手写尺寸统一以 `vp` 为基准单位（CLAUDE.md §6 基准画布 1280×800 vp）。
  - `exercise_blanks.width_em` 的 `em` 是**相对练习正文字号**的倍数；固定 `1em = 16vp`（练习正文基准字号），换算 `emToVp(em) = em × 16`。任何把 `widthEm` 与 `vp` 比较/取值的表达式必须先经 `emToVp` 换算。（`1em=16vp` 为可执行基准值，可随正文字号 token 调整，但必须给定值。）
  - 笔迹文件（§4.1）`canvas.width/height` 与坐标点 `x/y` 的「逻辑像素」即 **vp**（1 逻辑像素 = 1 vp）；`handwriting_records.canvas_width/canvas_height` 同为 vp。物理像素换算 `px = vp × densityPixels` 交渲染层，不落库。
  - 笔迹画布尺寸与 blank 渲染宽高同坐标系（均 vp），保证重放/缩放（P-F08）不错位。

失败处理：

1. 标记 `import_jobs.state = failed`。
2. 回滚未提交数据库事务。
3. 删除任务暂存目录。
4. 若已创建回滚点，恢复旧书籍目录。
5. 保留不含敏感内容的错误摘要。

### 2.4 校验顺序

1. 文件头和 MIME 确认为 ZIP。
2. 枚举条目，不解压即检查路径、条目数、大小、压缩比。
3. 读取并校验 `manifest.json`。
4. 检查协议主版本兼容性。
5. 校验必需文件和允许扩展名。
6. 解压到唯一隔离暂存目录，状态为 `extracting_to_staging`；该目录对书架不可见。
7. 对暂存目录内文件逐个计算 SHA-256 并比对 `checksums.json`。
8. 使用**应用内置** JSON Schema 校验暂存目录内的结构化文件：`manifest.json`、`book.json`、每个 `chapters/*.json`、`report.json`、`checksums.json` 分别对应 `packages/studybook-schema/` 的 manifest/book/chapter/report/checksums Schema（R-043）。**一律忽略包内自带的 `schemas/` 目录，绝不以不可信包内 Schema 校验自身**（封死信任边界，BR-002）。任一文件 Schema 校验失败记 IMP-008。
9. 校验目录、ID、文件引用、章节顺序、练习空位和答案。
10. 生成应用端校验报告并进入 `awaiting_confirmation`。
11. 用户确认后执行 `indexing`，在数据库事务中建立内容索引。
12. 索引成功后将暂存目录原子提升到正式目录，状态为 `promoting_to_final`。
13. 提交数据库事务并进入 `completed`。

`awaiting_confirmation` 之前的“解压”只写入任务隔离暂存区，不属于安装到书架；
确认后不得再次解压。数据库提交失败或目录提升失败时按失败处理规则清理正式目录并回滚。

### 2.5 导入错误码

| 错误码 | 含义 | 是否阻断 |
|---|---|---|
| IMP-001 | 不是有效 ZIP | 是 |
| IMP-002 | ZIP 路径穿越 | 是 |
| IMP-003 | 条目数量或大小超限 | 是 |
| IMP-004 | 疑似压缩炸弹 | 是 |
| IMP-005 | 缺少 manifest.json | 是 |
| IMP-006 | 协议主版本不兼容 | 是 |
| IMP-007 | JSON 解析失败 | 是 |
| IMP-008 | Schema 校验失败 | 是 |
| IMP-009 | SHA-256 不一致 | 是 |
| IMP-010 | 内部文件引用缺失 | 是 |
| IMP-011 | 章节 ID/顺序冲突 | 是 |
| IMP-012 | 可判分练习缺少答案 | 是 |
| IMP-013 | 存储空间不足 | 是 |
| IMP-014 | 重复版本 | 否，需用户选择 |
| IMP-015 | 存在待人工复核内容 | 否，警告 |
| IMP-016 | 数据库提交失败 | 是 |
| IMP-017 | 复制到暂存区失败（`copying` 阶段，非磁盘满的通用 IO 错误） | 是 |
| IMP-018 | 解压失败（`extracting_to_staging` 阶段，条目损坏/解码失败，区别于路径穿越 IMP-002 与压缩炸弹 IMP-004） | 是 |
| IMP-019 | 建索引失败（`indexing` 阶段，区别于 `committing` 提交失败 IMP-016） | 是 |
| IMP-020 | 存在协议九种之外的未知练习题型 | 是 |

> 错误码覆盖状态机全部 `failed` 入口（R-056）：`copying→failed`=IMP-017、`extracting→failed`=IMP-018、`indexing→failed`=IMP-019、`committing→failed`=IMP-016；磁盘空间不足在任意阶段统一记 IMP-013。`validating_business` 三类失败对应 IMP-010/011/012。所有 IMP-* 的用户可读文案来源见 §3.14A error_code→资源键映射表。

**IMP-015 触发数据源（R-037）**：导入完成时满足以下任一即置 IMP-015 非阻断警告（写入 `import_jobs.warning_count`，并按《[06-协议到数据库映射规范](./06-协议到数据库映射规范.md)》§8 落 `content_review_flags`，见 §3.25）：

- `report.json` 的 `reviewQueue` 非空；或
- 任一章节 `quality.reviewStatus='needs-review'`（落库为 `chapters.review_status='needs-review'`）；或
- `report.json` 的 `status='completed-with-warnings'`。

三处语义统一收敛为「该包存在待人工复核内容」，IMP-015 仅作非阻断提示、不影响导入成功。`report.json` 在 §2.4 第 5 步校验「存在」、第 8 步由 `report.schema.json` 校验结构（D4 新增），不在第 3 步（该步只校验 `manifest.json`）。

### 2.6 取消、重试与后台执行

**取消（R-014）**

- `selected`、`copying`、`validating_zip…validating_business`、`extracting_to_staging`、`awaiting_confirmation` 任一状态均可转入 `cancelled`（对应 FR-F02-05“写入数据库前可取消”）；`indexing` 及之后不可取消。
- 进入 `cancelled` 统一清理：删除 `temp/import/<jobId>` 暂存目录、回滚未提交事务、置 `import_jobs.state=cancelled` 并保留脱敏摘要；取消不写 `error_code`，不计入 `warning_count`。

**重试（R-015）**

- 失败区分**可重试**（环境/IO 类：IMP-013 存储不足、IMP-016 提交失败、IMP-017 复制失败、IMP-019 建索引失败）与**终态拒绝**（内容类：IMP-002 路径穿越、IMP-009 哈希不符、IMP-018 解压损坏等，不可重试）。
- 可重试失败允许 `failed → copying`，**复用同一 `jobId`**，清理上次暂存目录后重建。
- 为支持重试，`ImportService.create` 必须保存可重开的**持久化 URI 授权**（不能仅存 `source_uri_hash`）；终态拒绝或用户放弃后释放该授权。

**后台执行与重启恢复（R-017）**

- 导入由独立后台任务承载，状态机持久化于 `import_jobs`；页面只订阅进度，离开页面不中断（对应 HLD §5.3）。
- `awaiting_confirmation` 在用户离开后挂起等待，超过 24 小时未确认自动转 `cancelled` 并清理。
- 应用重启时扫描非终态 `import_jobs`：`indexing` 之前的恢复为可继续或按可重试失败处理；`indexing`/`promoting_to_final` 中断的回滚至 `failed` 并清理半成品，绝不残留半本书。

### 2.7 升级（upgrade）子流程（R-016）

`commit(jobId, 'upgrade')` 在 `indexing` 与 `committing` 之间插入回滚点与目录切换：

1. `snapshot_old`：将旧 `files/books/<bookId>` 重命名为回滚点。
2. 按 `external_id` 原地映射写入新内容（见《[06-协议到数据库映射规范](./06-协议到数据库映射规范.md)》§10）。
3. `promoting_to_final`：原子切换到正式目录。
4. 任一步失败 → `restore_old`：删除新目录、恢复回滚点、回滚事务，`books.status` 回到 `ready`。

- 升级期间 `books.status='updating'`，成功后置 `ready`；书架对 `updating` 的书显示“更新中”，不可进入学习。
- 同一 `package_id` 同时只允许一个导入/升级任务（见 §8）。

#### 3.12.1 完成度可执行公式

仅 `kind IN ('lesson','review')` 的章节参与学习完成度；`appendix`、`answer-key`、
`index` 不进入书籍完成度分母。

定义：

```text
S = 章节内需要学习的 sections 数量
Sr = 已读 sections 数量
E = enabled=1 的 exercises 数量
Es = 已提交且 grade ∈ {correct, partial, incorrect} 的 exercises 数量
R = S=0 时为 1，否则 Sr/S
P = E=0 时为 1，否则 Es/E
```

“已读 section”满足以下任一条件：

- 用户滚动到该 section 最后一个内容块可视区域的 80% 以上；或
- 用户从该 section 进入后续 section/练习，且该 section 前台停留累计不少于 3 秒。

章节完成度：

```text
若 S>0 且 E>0：chapterCompletion = 100 × (0.4R + 0.6P)
若 S>0 且 E=0：chapterCompletion = 100 × R
若 S=0 且 E>0：chapterCompletion = 100 × P
若 S=0 且 E=0：chapterCompletion = 0
```

- 计算结果保留 1 位小数，使用四舍五入。
- 每次 section 阅读状态或 exercise 提交状态变化后，由
  `ProgressService.recomputeChapter` 全量覆盖 `completion_percent`，不做累加。
- `completion_percent=100.0` 时设置 `completed_at`；低于 100 时清空 `completed_at`。

书籍完成度：

```text
C = kind IN ('lesson','review') 的章节集合
bookCompletion = C 为空时为 0，否则 sum(chapterCompletion) / |C|
```

书架、目录和统计必须调用同一 `ProgressCalculator`，不得分别计算。

#### 3.12.2 章节状态判定（R-026）

`reading_progress.status` 四态（FR-F04-03）由以下可执行条件派生，与 `completion_percent` 同源、由 `ProgressService.recomputeChapter` 在每次进度变化后一并写入，页面不得自行判定：

| status | 进入条件（优先级自上而下，命中即止） |
|---|---|
| review-due | 该章存在 `wrong_items.next_due_at ≤ now`（`status='active'`）或 `review_cards.due_at ≤ now`（`state≠'new'`）的到期项 |
| completed | `completion_percent = 100.0`（所有需学习 section 已读且所有 `enabled` 练习均已提交一次） |
| learning | `0 < completion_percent < 100` |
| not-started | `completion_percent = 0` 且无任何阅读/作答记录 |

- `review-due` 是覆盖态：到期项存在时优先于 `completed`/`learning`，到期项清空（复习完成或错题 `mastered`）后回落到由 `completion_percent` 决定的态。
- `completed` 不要求练习全部判对：提交即计入完成度（与 §3.12.1 `Es` 口径一致），「已完成」指学习动作完成；「掌握章节数」（FR-F11-01）另由掌握度口径计算（见 §3.13A），二者不混用。
- 到期真值源固定：错题巩固=`wrong_items.next_due_at`、间隔复习=`review_cards.due_at`；`status='review-due'` 仅为二者派生展示态，不独立存储到期时间。

#### 3.13.1 连续天数与有效学习时长口径（R-027）

时间列统一存 UTC（§1.3），但「连续天数」是本地日历概念，故定义如下底层口径（上层 `StatsResult` 指标见 §3.13A）：

- **切日时区**：`streak` 按**设备当前本地时区**把 `event_at` 归入自然日（`floor((event_at + tzOffsetMs) / 86400000)`）。跨夏令时/改时区时以事件发生当时的本地偏移为准，已计入的历史自然日不回算。
- **「有效一天」门槛**：某本地自然日内**有效学习时长 ≥ 5 分钟**或**完成 ≥ 1 道练习提交**（`grade∈{correct,partial,incorrect}`），该日才计入 streak；仅打开未学习不计入。（5 分钟为可执行基准值，可随设计 token 微调，但必须给定值。）
- **有效学习时长聚合**：累加 `event_type='active'` 段的 `duration_ms`；`pause` 后到下一个 `active` 前的空窗不计入；进入 `pause` 前的 2 分钟无操作空闲不计入（以最后一次有效操作时间封口）。
- 单次 `duration_ms` ≤ 10 分钟，更长会话已拆为心跳事件，聚合时直接求和、不二次去重。
- 以上由 `StatisticsService`（04 契约 §9）实现，FR-F11「误差不超过 1 分钟」以此为准。

## 4. 手写数据与识别设计

### 4.1 笔迹文件格式

文件扩展名：`.stroke.json.gz`。

```json
{
  "format": "ai.studybook.stroke",
  "version": 1,
  "canvas": {"width": 320, "height": 96},
  "strokes": [
    {
      "id": "stroke-1",
      "tool": "pen",
      "color": "#2F65A4",
      "width": 3,
      "points": [
        {"x": 12.4, "y": 50.1, "t": 0, "p": 0.42, "tiltX": 0, "tiltY": 0}
      ]
    }
  ]
}
```

约束：

- 坐标以画布逻辑像素保存；逻辑像素即 `vp`（1 逻辑像素 = 1 vp，见 §1.3 单位换算 R-035），与 blank/画布同坐标系（故下条「32 vp」与本坐标同一单位）。
- `t` 为相对首点毫秒数且单调不减。
- 压力 `p` 范围 0..1；设备不支持时使用 0.5。
- 不保存超出画布 32 vp 以上的点。
- 页面退出前强制落盘；书写中每 2 秒保存恢复快照。
- 进入练习书写即创建 `grade='draft'`、`recognition_status='not-requested'` 的 `exercise_attempts` 行作为快照锚；每 2 秒快照写入 `handwriting/<bookId>/<chapterId>` 临时 stroke 文件并更新 `handwriting_records`（R-021）。
- 判分成功或用户放弃后清理未提交 draft；应用重启时扫描 `grade='draft'` 行恢复画布，超出保留期的孤儿 draft 由后台清理。

### 4.2 识别状态机

```text
idle → writing → ready
ready + 用户点击“确认答案” → recognizing
recognizing → grading → correct / partial / incorrect
recognizing → recognition-failed → retry / rewrite
```

约束：

- 停笔只更新本地草稿，不自动调用识别服务。
- “确认答案”在识别请求处理中禁用，防止重复提交。
- 界面不展示置信度和候选词。
- 识别服务必须通过
  [《手写识别验收规范》](./05-手写识别验收规范.md)：
  `MX-HWR-EN-2000-v1` 固定测试集空位精确识别准确率至少 90%。
- 服务失败、超时或返回空文本时进入 `recognition-failed`，不执行判分；并据情形写入 `recognition_error_code`（超时=`timeout`、空文本=`empty-text`、服务不可用=`service-unavailable`、模式不支持=`unsupported-mode`、其它=`internal-error`），retry/rewrite 分支据码区分提示（R-057，取值集见 §3.14/04 契约 §5）。
- 识别硬超时 5 秒（与《[05 手写识别验收规范](./05-手写识别验收规范.md)》一致），由识别服务层计时，超时触发 `RecognitionHandle.cancel()` 后进入 `recognition-failed`；“确认答案”禁用在超时后解除，允许重试/重写，不死锁（R-024）。
- 应用重启时 `recognition_status='processing'` 的 attempt 复位为 `not-requested`，不留挂起态。
- 多空题（R-018）：各空独立保存 `recognized_text_json`/`normalized_text_json` 与逐空判分（04 契约 `perBlank`）；整题 `recognition_status` 由各空聚合（全 `succeeded`→`succeeded`，含失败→`failed`）。存在 `recognition-failed` 空时整题不产生最终 `grade`，仅标记该空待重写并保留其余空的识别与笔迹，待该空重写后整体判分，避免因一个空失败而误判整题（满足 FR-F07-09）。

### 4.3 答案标准化

按以下顺序：

1. Unicode NFC。
2. 去除首尾空白。
3. 连续空白压缩为一个空格。
4. 弯引号统一为直引号。
5. 根据题目配置决定是否忽略大小写。
6. 根据题目配置决定是否忽略末尾句号、问号或感叹号。
7. 保留内部撇号，不将 `is` 和 `'s` 无条件互换。

### 4.4 判分优先级

1. 识别服务返回的文本。
2. 主答案精确匹配。
3. 可接受答案精确匹配。
4. 学习包 `grading_rules` 按 `priority` 执行。
5. 内置语法规则（见下表）。
6. 拼写/识别相似提示。
7. 错误。

**内置语法规则集（R-028）**：仅在学习包 `grading_rules` 未命中时按下序执行，其 `feedback_code` 取《[06-协议到数据库映射规范](./06-协议到数据库映射规范.md)》§5 受控词表（非来自学习包）：

| 序 | 规则 | 命中结果 | feedback_code |
|---|---|---|---|
| 5.1 | 与某 accepted 仅时态不同（base/过去/进行时互换） | partial | wrong-tense |
| 5.2 | 与某 accepted 仅词形不同（单复数、第三人称单数 -s） | partial | wrong-form |
| 5.3 | 缺少/多出一个 token（助动词、冠词等） | partial | missing-word / extra-word |
| 5.4 | token 顺序错误但集合一致 | partial | word-order |

**编辑距离阈值语义（`gradingOptions.maxEditDistance`，R-028）**：

- 与任一 accepted 精确匹配 → `correct`（第 2/3 步）。
- 非精确匹配、但编辑距离 ≤ `maxEditDistance` 且未命中上述任一 partial 规则 → 判 **`incorrect`** 并叠加「可能存在识别或拼写误差」提示文案；**不改变 `grade`、不计 `partial`、不改 `score`**（编辑距离只产生提示，与协议 §6.2 一致）。
- 命中内置 partial 规则的，按规则结果（partial）优先于编辑距离提示。

编辑距离只能产生“可能存在识别或拼写误差”的错误提示，不得直接产生 `correct`。

### 4.5 判分结果

| 结果 | score | 行为 |
|---|---:|---|
| correct | 1.0 | 更新进度，正常进入下一题 |
| partial | 0.5 | 显示具体缺失，加入错题 |
| incorrect | 0.0 | 显示解释，加入错题 |
| recognition-failed | NULL | 不判分，保留笔迹并允许重试或重写 |

多空题整题结果由各空 `BlankGradeResult` 聚合（04 契约 §12）：全部 `correct` → `correct`；存在 `incorrect` → `incorrect`；否则 `partial`。整题 `score` 为各空 `score` 算术平均；任一空 `recognition-failed` 时整题不落最终 `grade`（见 §4.2）。

## 5. 复习调度设计

评分：

```text
0 = 忘记了
1 = 有点难
2 = 掌握
3 = 太简单
```

第一版调度规则：

- 新卡首次评分 0：10 分钟后。
- 新卡首次评分 1：1 天后。
- 新卡首次评分 2：3 天后。
- 新卡首次评分 3：7 天后。
- 已复习卡：
  - 0：`interval = 0`，`lapses + 1`，当天稍后再次出现。
  - 1：`interval = max(1, round(oldInterval * 1.2))`。
  - 2：`interval = max(2, round(oldInterval * easeFactor))`。
  - 3：`interval = max(4, round(oldInterval * easeFactor * 1.3))`。
- easeFactor 根据评分调整，但限制在 1.3..3.5。

所有调度变化必须记录 `review_logs`，允许回放和调试。

### 6.13 路由参数约定（R-061）

- **来源标识 `from`**：P-F06/P-F07 复用同一练习路由，新增可选 `from` 区分进入来源——
  `from=chapter`（默认，缺省按此处理）由章节讲解/目录进入，「返回」回到该章节；
  `from=wrongbook` 由错题本「重做」进入，「返回」回到错题本列表并保留其筛选状态。
  判分页 P-F07 作为 P-F06 的子状态，沿用同一 `from`，不新增独立返回栈。
- **可选 `bookId`**：`review?bookId={optional}`、`wrongbook?bookId={optional}` 中 `bookId`
  **省略 = 全部书籍聚合**，传入则限定该书。
- **`range` 合法值与兜底**：`statistics?range={day|week|month}`，合法集合为 `day`/`week`/`month`，
  缺省 `week`；遇非法值回退 `week`，不取空数据。
- 深链恢复遵循 HLD『深层页面返回时恢复上层列表状态』：按 `from`/`bookId`/`range` 还原上层入口与筛选。

> 说明：评审 R-061 原列的「`range` 缺省值缺失」「lesson `section` 缺省未写」经核已分别由 §6.11
> （`range=week`）与 FR-F04-05（默认小节）覆盖，本次只补来源标识与可选 query 的省略/非法语义。

## 8. 并发与事务

- 同一个 `packageId` 同时只能存在一个导入/升级任务。
- 同一练习同时只允许一个活动识别请求；新请求取消旧请求。
- 笔迹落盘串行化，避免同一文件并发覆盖。
- 导入数据库写入使用一个事务。
- 文件原子移动成功但数据库提交失败时必须删除新目录或恢复旧目录。
- 设置采用最后写入生效，但写入必须经过键级校验。

### 8.1 数据生命周期操作（R-023）

**删除模型（R-038）**

`books` 是**唯一软删除锚点**，软删除与物理 `CASCADE` 两套语义不再含糊，由 BR-010 用户选择二选一：

1. **保留学习记录（软删除）**：置 `books.status='deleted'` + `deleted_at`，章节/练习/进度/错题/复习卡**全部保留**；书架、继续学习、统计、复习等所有入口按 §1.3「查询默认排除已删除」过滤其属主书而隐藏这些记录（下游表无需各自加 `deleted_at`）。按用户选择可单独清理笔迹文件。
2. **彻底删除（物理）**：在一个数据库事务内物理 `DELETE` `books` 行，`ON DELETE CASCADE` 连带清除 `chapters`/`exercises`/`exercise_attempts`/`wrong_items`/`review_cards`/`reading_progress`/`notes`/`highlights`/`bookmarks` 等全部下游行。

> 这样消除评审 R-038 指出的两点风险：软删书后其 attempts/事件**因属主过滤不再参与统计**（不虚高）；彻底删除前经二次确认且可「随备份导出」（FR-F10-06 保留历史、SRS 删除可选保留），不会无声清空用户练习历史。
> 仅 `notes`/`highlights` 额外自带 `deleted_at`，用于**批注级**软删除与升级重定位（§3.16/§3.17），与书级删除模型正交。

**删除书籍流程**

- 两阶段：先在一个数据库事务内按 BR-010 用户选择执行上述软删除或物理删除（彻底删除时连带 `exercise_attempts`、`wrong_items`、`notes`、`highlights`、`review_cards`、`reading_progress`），再删除文件仓 `books/<bookId>`、`handwriting/<bookId>`、`notes/<bookId>` 与对应 `derived/`；文件删除失败可重试，不阻塞已提交的 DB 事务。
- BR-010“是否同时删除笔迹与学习记录”作为删除参数控制级联范围；删除前二次确认并说明影响。

**导出 / 恢复**

- `RestoreMode='merge'`：按 `package_id + content_version` 合并，冲突时保留本机较新版本；`'replace-all'`：清空现有数据后整体恢复。
- 恢复前由 `BackupService.validateBackup` 校验版本与完整性（FR-F12-05），不兼容则拒绝并报告。

**清理缓存**

- 白名单仅 `cache/images`、`cache/recognition` 与可重建的 `derived/`；正文、笔迹、进度、错题不属于缓存，不得删除（FR-F12-06）。

## 9. 数据升级

数据库迁移命名：

```text
V001__initial.sql
V002__add_review_tables.sql
V003__add_content_version.sql
```

规则：

- 每个迁移只执行一次。
- 迁移前备份数据库元数据。
- 禁止在生产迁移中使用“删除所有表再重建”。
- 学习包内容升级按 `package_id + content_version` 判断。
- 升级时通过 `external_id` 映射旧章节、练习和批注锚点。
- 升级采用按 `external_id` **原地 UPDATE**，保持内部主键稳定，从而 `exercise_attempts`/`wrong_items`/`review_cards` 的外键与冗余列（`book_id`/`chapter_id`/`exercise_id`）自然保持一致，历史答题与错题不丢失（R-006，见《06-协议到数据库映射规范》§10）。
- 新包缺失的旧 `external_id`（被删练习）：对应 `exercises.enabled=0` 并保留其 attempts/wrong_items，不物理删除。
- 无法映射的笔记保留并标记 `needs-relocation`。
- **阅读进度锚点重映射（R-048）**：升级时同步按 `external_id` 重映射 `reading_progress.last_section_id`/`last_block_id` 到新行；无法映射时**回退到 `scroll_anchor_json` 或章节首**，不得直接置 NULL，以免「继续学习」定位静默丢失（FR-F05-04/FR-F01-03）。`reading_progress` 自身不加 `deleted_at`，其随软删书的隐藏由 §1.3 属主过滤覆盖。
- **冗余列同步清单（R-047）**：每次导入/升级提交事务内必须重算/校准的反规范化列——
  `books.chapter_count`（`SELECT COUNT(*)` 重算，§3.3）、`exercise_attempts.book_id`/`chapter_id`（按 `external_id` 原地 UPDATE 随 `exercises.chapter_id` 同步，声明为「只读快照、由升级映射回填」，不允许与 `exercises` 漂移）。
