## 1. 文档目的

本文定义“墨学书架”第一版的产品范围、功能需求、数据要求、非功能需求和验收标准。
开发、测试和验收必须使用本文中的需求编号进行追踪。

## 2. 产品目标

用户将 PDF 交给 Codex、Claude 等外部 AI 工具，AI 按 `ai.studybook` 协议生成一个
`.studybook.zip`。应用导入学习包后，用户无需再打开 PDF，即可：

- 在一个书架中管理多本结构化学习书。
- 按书籍、章节、小节学习内容。
- 使用手写笔在横线上完成英语练习。
- 将笔迹识别为文本，并进行语法感知判分。
- 保存笔记、高亮、错题、复习计划和学习进度。
- 在没有网络的情况下完成日常学习。

## 3. 产品范围

### 3.1 第一版范围内

- `.studybook.zip` 导入、安全校验和本地安装。
- 多书籍书架。
- 章节目录、讲解、例句、表格和原创插图展示。
- 手写填空、短答和改写练习。
- 第一版练习仅支持需要手写输入的题型；选择、判断、连线和词序拖拽不属于第一版。
- 用户确认后执行英文手写识别与语法判分。
- 手写笔记、高亮、收藏。
- 复习卡片、错题本和学习统计。
- 本地设置、数据导出和缓存管理。

### 3.2 第一版范围外

- 在鸿蒙应用内直接解析 PDF 或执行整本 OCR。
- 账号、云同步、社区、班级和教师后台。
- 在线商城、付费订阅和版权内容分发。
- 多人共享书架。
- 自动修改或回传原始学习包。
- 通用任意语言手写识别；第一版重点支持英文单词和短句。

## 4. 用户与使用环境

### 4.1 用户角色

第一版只有“本机学习者”一个角色。用户拥有导入、学习、批注、删除和导出本机数据的权限。

### 4.2 使用环境

- HarmonyOS 平板，横屏为主要使用方式。
- 支持触控和系统兼容手写笔。
- 日常学习可以完全离线。
- 学习包由电脑端或其他 AI 工具生成并传入平板。

## 5. 核心业务规则

| 编号 | 规则 |
|---|---|
| BR-001 | 应用不接收 PDF，只接收 `.studybook.zip`。 |
| BR-002 | 未通过安全、Schema、哈希和引用完整性校验的学习包不得进入书架。 |
| BR-003 | 导入失败不能产生半本书；所有临时文件和数据库写入必须回滚。 |
| BR-004 | 章节切换不得以左右滑动作为唯一入口，避免与书写手势冲突。 |
| BR-005 | 手写墨迹、识别文本和判分结果必须分别保存。 |
| BR-006 | 用户点击“确认答案”后，识别服务直接识别并判分；界面不展示置信度和候选词。 |
| BR-007 | 判分必须依据学习包答案与规则，不能仅依赖字符串相似度。 |
| BR-008 | 原书扫描页不能作为学习正文；正文必须是结构化数据。 |
| BR-009 | 学习数据默认仅保存在本机，不得未经许可上传。 |
| BR-010 | 删除书籍前必须二次确认，并说明是否同时删除笔迹与学习记录。 |

## 6. 功能需求

### 6.1 F01 我的学习书架

![F01 我的学习书架](../assets/mockups/F01-bookshelf.png)

**需求：**

- FR-F01-01：展示全部已导入书籍的封面、标题、作者、学习进度和继续学习入口。
- FR-F01-02：展示正在导入、导入失败和内容损坏状态。
- FR-F01-03：用户点击书籍卡片进入书籍目录；点击“继续学习”恢复上次位置。
- FR-F01-04：支持按最近学习、标题和导入时间排序。
- FR-F01-05：空书架必须展示导入引导。
- FR-F01-06：书籍数量增加时支持滚动，不允许卡片互相覆盖。

**验收：**

- 导入成功后 1 秒内出现在书架。
- 应用重启后书籍顺序和进度保持一致。
- 内容损坏的书籍不可进入学习页，并提供修复或删除入口。

### 6.2 F02 导入学习包

![F02 导入学习包](../assets/mockups/F02-import.png)

**需求：**

- FR-F02-01：通过系统文件选择器选择一个 `.studybook.zip`。
- FR-F02-02：显示文件名、大小和来源，不显示完整敏感路径。
- FR-F02-03：只允许一次导入一个学习包。
- FR-F02-04：导入开始前检查可用存储空间。
- FR-F02-05：支持用户在写入数据库前取消操作。
- FR-F02-06：不接受 PDF、RAR、7z 或改扩展名的非 ZIP 文件。

**验收：**

- 错误扩展名或 MIME 类型会被明确拒绝。
- 文件选择取消后返回原页面且不产生临时数据。
- 存储不足时显示所需空间和当前可用空间。

### 6.3 F03 学习包校验与导入

![F03 学习包校验](../assets/mockups/F03-validation.png)

**需求：**

- FR-F03-01：依次执行 ZIP 安全、协议版本、Schema、SHA-256、文件引用、目录章节和练习答案校验。
- FR-F03-02：展示当前阶段、总体进度和可读的失败原因。
- FR-F03-03：区分阻断错误与非阻断警告。
- FR-F03-04：校验通过后由用户确认才写入书架。
- FR-F03-05：重复导入同一 `packageId + contentVersion` 时提示已存在。
- FR-F03-06：新版本可选择覆盖内容并保留学习记录；覆盖前必须生成回滚点。
- FR-F03-07：导入日志只保存错误码和必要上下文，不记录完整正文。

**验收：**

- 包含 `../` 路径、绝对路径或压缩炸弹特征的 ZIP 被拒绝。
- 任一必需 JSON 解析失败时整包拒绝。
- 导入中断后，书架和数据库中不存在半成品记录。

### 6.4 F04 书籍详情与章节目录

![F04 书籍目录](../assets/mockups/F04-catalog.png)

**需求：**

- FR-F04-01：展示书籍资料、总章节数、完成度和最近学习位置。
- FR-F04-01A：完成度必须由统一可重算公式生成：同时含讲解和练习的章节，
  讲解阅读占 40%、练习提交占 60%；无练习章节按讲解阅读占 100%；
  书籍完成度为全部 `lesson/review` 章节完成度的算术平均值。
- FR-F04-02：按学习包目录顺序展示章节。
- FR-F04-03：章节显示未学习、学习中、已完成、需复习状态。
- FR-F04-04：支持按章节标题、中文标题和关键词搜索。
- FR-F04-05：点击章节进入其默认小节。
- FR-F04-06：附录、答案、索引与普通课程使用不同标识。

**验收：**

- 115 个章节可以在目录中完整显示并定位。
- 搜索“现在进行时”能够找到对应章节。
- 返回目录时保持原滚动位置。
- 相同原始阅读与作答记录在书架、目录和统计页计算出的完成度必须一致，误差为 0。

### 6.5 F05 章节讲解

![F05 章节讲解](../assets/mockups/F05-lesson.png)

**需求：**

- FR-F05-01：渲染段落、公式、例句、列表、表格、时间线、提示框和插图。
- FR-F05-02：提供讲解 A、讲解 B、练习、复习等章节内标签。
- FR-F05-03：提供目录、上一章和下一章按钮。
- FR-F05-04：自动保存当前章节、小节和阅读位置。
- FR-F05-05：支持收藏当前章节或内容块。
- FR-F05-06：图片缺失时显示替代文本，不阻断文字学习。
- FR-F05-07：不显示原 PDF 页，来源页码只用于内部追溯。

**验收：**

- 表格和公式在横屏基准宽度下无横向截断。
- 退出并重新进入后恢复到最近阅读位置。
- 快速切换章节时不会显示上一章节的残留内容。

### 6.6 F06 手写练习

![F06 手写练习](../assets/mockups/F06-handwriting.png)

**需求：**

- FR-F06-01：题目由文字段和一个或多个可书写横线组成。
- FR-F06-02：笔迹必须跟随手写笔即时显示。
- FR-F06-03：支持撤销、重做、清除当前空和清除整题。
- FR-F06-04：每个空位有独立书写区域和答案。
- FR-F06-05：停笔后只保存当前笔迹，不自动触发识别或判分。
- FR-F06-06：手掌触碰不得产生笔迹；手写笔和触控点击要区分。
- FR-F06-07：未书写时不得提交，除非题型允许空答案。
- FR-F06-08：用户点击“确认答案”后，系统一次性提交笔迹并触发识别与判分。
- FR-F06-09：协议保留全部九种题型。`fill-blank-handwriting`、`multi-blank-handwriting`、
  `rewrite-sentence-handwriting`、`short-answer-handwriting`、`correction-handwriting` 五种手写题型
  在第一版完整实现；`choice`、`true-false`、`match`、`order-words` 四种非手写题型导入后保留并占位展示，
  作答与判分交互列入后续迭代。

**验收：**

- 墨迹显示延迟 P95 不高于 20 ms。
- 横线有效书写高度不低于 88 vp。
- 书写跨越横线边界时不会污染相邻题目。
- 学习包包含第一版白名单外题型时必须在导入业务校验阶段阻断，不允许导入后才显示不可作答页面。

### 6.7 F07 手写识别与语法判分

![F07 识别与判分](../assets/mockups/F07-grading.png)

**需求：**

- FR-F07-01：用户点击“确认答案”后，后台识别服务直接识别手写内容并执行判分。
- FR-F07-02：正常流程不展示识别文本、置信度或候选词，也不要求用户二次确认识别结果。
- FR-F07-03：执行大小写、空格、弯直引号和末尾标点标准化。
- FR-F07-04：标准答案、可接受答案和部分正确规则来自学习包。
- FR-F07-05：支持正确、部分正确、错误、待确认四种结果。
- FR-F07-06：反馈应说明具体错误，例如“缺少助动词 is”。
- FR-F07-07：拼写相似只能触发“可能识别错误”提示，不能自动判正确。
- FR-F07-08：保存原始笔迹、服务识别文本、归一化答案和判分结果，用于错题与问题追踪。
- FR-F07-09：识别服务无法返回有效文本时不得产生错误判分，应保留笔迹并提示重试或重写。

**验收：**

- `is drinking` 判正确。
- 仅写 `drinking` 时按规则判部分正确并提示缺少 `is`。
- 服务识别结果为 `is thinking` 时不能因编辑距离接近而判正确。
- 后台识别服务必须按
  [《手写识别验收规范》](./05-手写识别验收规范.md)执行固定数据集测试，
  空位精确识别准确率达到 90% 以上。

### 6.8 F08 笔记、高亮与收藏

![F08 笔记与批注](../assets/mockups/F08-notes.png)

**需求：**

- FR-F08-01：支持手写笔记、文本笔记、高亮、下划线和收藏。
- FR-F08-02：批注与正文分层保存，不能改写学习包原内容。
- FR-F08-03：笔迹锚定到章节和内容块。
- FR-F08-04：支持撤销、重做、橡皮擦和颜色/粗细设置。
- FR-F08-05：批注应自动保存，并显示保存状态。
- FR-F08-06：内容升级后无法重新定位的批注进入“待重新定位”状态。

**验收：**

- 应用异常退出后，最近一次已落盘笔迹可以恢复。
- 删除高亮不会删除正文。
- 同一章节的笔记可以从“我的笔记”再次定位。

### 6.9 F09 复习卡片

![F09 复习卡片](../assets/mockups/F09-review.png)

**需求：**

- FR-F09-01：展示今日到期卡片数量。
- FR-F09-02：支持显示答案前手写作答。
- FR-F09-03：用户可选择忘记、有点难、掌握、太简单。
- FR-F09-04：系统据此计算下次复习日期。
- FR-F09-05：卡片关联原书籍、章节和知识点。
- FR-F09-06：完成后展示本次复习摘要。

**验收：**

- 评分后卡片的到期时间立即更新。
- 同一张卡不会在一次复习会话中无理由重复。
- 可以从卡片返回原章节。

### 6.10 F10 错题本

![F10 错题本](../assets/mockups/F10-wrongbook.png)

**需求：**

- FR-F10-01：错误和部分正确答案自动进入错题本。
- FR-F10-02：按书籍、章节、语法点、错误类型和时间筛选。
- FR-F10-03：展示用户答案、正确答案和错误原因。
- FR-F10-04：记录重复错误次数。
- FR-F10-05：支持单题重做和批量错题练习。
- FR-F10-06：连续正确达到配置阈值后标记为已掌握，但保留历史记录。

**验收：**

- 新错题在判分完成后立即可见。
- 重做后历史错误次数不被覆盖。
- 删除书籍时对应错题按用户选择删除或随备份导出。

### 6.11 F11 学习统计

![F11 学习统计](../assets/mockups/F11-statistics.png)

**需求：**

- FR-F11-01：统计学习时长、练习数、正确率、连续天数和掌握章节数。
- FR-F11-02：支持日、周、月维度。
- FR-F11-03：按书籍展示掌握度和薄弱知识点。
- FR-F11-04：不将页面停留但无操作的长时间计入有效学习时长。
- FR-F11-05：统计结果可从原始学习事件重新计算。

**验收：**

- 完成练习后统计在 2 秒内更新。
- 切换时间范围时数据显示一致且不重复累计。
- 学习时长与事件记录抽样核对误差不超过 1 分钟。

### 6.12 F12 设置与数据管理

![F12 设置与数据管理](../assets/mockups/F12-settings.png)

**需求：**

- FR-F12-01：设置墨迹颜色、粗细和识别失败后的默认处理方式；不提供自动识别与候选词设置。
- FR-F12-02：设置复习提醒和每日目标。
- FR-F12-03：展示书籍、图片、笔迹和缓存占用空间。
- FR-F12-04：支持导出学习数据备份。
- FR-F12-05：支持从兼容备份恢复，恢复前校验版本和完整性。
- FR-F12-06：清理缓存不能删除书籍正文、笔迹或学习记录。
- FR-F12-07：删除全部数据必须二次确认。

**验收：**

- 设置修改后立即生效并在重启后保留。
- 导出文件不包含原始 PDF。
- 清理缓存后离线正文仍可打开。

## 7. 学习包数据需求

### 7.1 必需文件

```text
manifest.json
book.json
report.json
checksums.json
chapters/*.json
assets/**/*
schemas/*.json
```

### 7.2 强制约束

- ZIP 根目录不能额外套同名文件夹。
- JSON 必须为 UTF-8、无 BOM、可由标准解析器读取。
- 所有内部路径必须是相对路径，分隔符使用 `/`。
- 章节 ID、练习 ID、空位 ID 在对应作用域内唯一。
- 每道可判分练习必须有标准答案、可接受答案或人工判分声明。
- 每个章节必须包含来源页引用和 OCR/复核状态。
- 不得包含原 PDF、可执行文件、脚本和路径穿越条目。

## 8. 非功能需求

### 8.1 性能

| 编号 | 指标 |
|---|---|
| NFR-P-001 | 冷启动至书架可操作 P95 ≤ 2.5 秒。 |
| NFR-P-002 | 已安装章节切换 P95 ≤ 300 ms。 |
| NFR-P-003 | 手写墨迹显示延迟 P95 ≤ 20 ms。 |
| NFR-P-004 | 用户点击确认后，单个英文短语识别与判分 P95 ≤ 1.5 秒。 |
| NFR-P-005 | 100 MB 学习包校验与导入在目标设备 P95 ≤ 60 秒。 |
| NFR-P-006 | 书架 100 本书时首页滚动保持流畅。 |
| NFR-P-007 | 后台识别服务在 `MX-HWR-EN-2000-v1` 固定测试集上的空位精确识别准确率 ≥ 90%，判定公式与环境见《手写识别验收规范》。 |
| NFR-P-008 | 协议上限（500 MB）学习包校验与导入在目标设备 P95 ≤ 240 秒；100–500 MB 区间导入时延随体量近似线性，超时须可取消并保持事务可回滚（R-072）。 |

### 8.2 可靠性

- NFR-R-001：导入和升级必须支持事务回滚。
- NFR-R-002：数据库升级不得清空用户数据。
- NFR-R-003：笔迹书写期间至少每 2 秒生成一次可恢复快照。
- NFR-R-004：关键数据写入失败必须提示用户，不能假装保存成功。
- NFR-R-005：离线识别模型不可用时必须降级为「保留笔迹 + 停用判分」，不得使主流程死锁或崩溃（R-065，见 HLD §3.3.1）。

### 8.3 安全与隐私

- NFR-S-001：默认不联网、不上传内容和笔迹。
- NFR-S-002：防止 ZIP Slip、压缩炸弹、超大单文件和伪造扩展名。
- NFR-S-003：导入包视为不可信输入。
- NFR-S-004：导出学习数据前显示包含内容并由用户确认。
- NFR-S-005：日志不得包含完整题目答案、笔迹数据和用户文件绝对路径。

### 8.4 可用性与无障碍

- NFR-A-001：主要触控目标不小于 48 × 48 vp。
- NFR-A-002：颜色不是状态的唯一表达方式。
- NFR-A-003：正文支持字体缩放。
- NFR-A-004：图片必须提供替代文本。
- NFR-A-005：手写主流程不要求调出软键盘。

### 8.5 兼容性

- NFR-C-001：学习包协议采用语义化版本。
- NFR-C-002：应用必须明确维护支持的主版本白名单。
- NFR-C-003：未知主版本拒绝导入；未知可选字段应忽略。

## 10. 总体验收条件

1. 一个包含 115 个章节的有效学习包可以成功导入。
2. 12 个功能页面均可从正常导航到达。
3. 用户可以完成“打开书籍 → 切换章节 → 手写答案 → 点击确认 → 后台识别并判分 → 进入错题/复习”的完整闭环。
4. 飞行模式下，已导入书籍的全部核心功能可用。
5. 应用重启后书籍、进度、笔迹、错题和设置不丢失。
6. 非法 ZIP、损坏 JSON、错误哈希和未知主版本均被安全拒绝。
